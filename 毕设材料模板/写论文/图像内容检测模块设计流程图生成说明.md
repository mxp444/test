# 3.3.4 图像内容检测模块设计流程图生成说明

## 使用位置

建议用于论文第 3 章第 3.3.4 节“图像内容检测模块设计”。该节需要说明系统如何对舆情帖子中的配图进行识别和质量评估，并将图像侧特征作为多模态融合风险评估的重要输入。

根据项目代码结构，图像内容检测模块主要由 `整合` 目录下 5 个以 `pic` 开头的 Python 文件构成：

| 文件名 | 模块作用 |
|---|---|
| `picOCR文字识别.py` | 提取图片中的文字内容及识别结果文件 |
| `pic二维码检测.py` | 判断图片中是否存在二维码，并返回二维码位置 |
| `pic模糊度检测.py` | 计算图片清晰度或边缘强度分数 |
| `pic色彩丰富程度.py` | 计算图片色彩、亮度、对比度等综合质量分数 |
| `pic设计感.py` | 基于 MobileNet 模型评估图片设计感或美学得分 |

## 一、图像内容检测模块流程图设计

### 流程图核心逻辑

图像内容检测模块以舆情帖子中的图片 `pics` 作为输入。系统从 MongoDB 文档中读取图片路径或图片链接，选择可用图片后执行图像侧检测。为了体现模块设计细节，流程图应重点展示五类图像检测子模块各自的工作路径：

1. 统一输入：帖子配图、本地图片路径或图片二进制数据。
2. 图像读取与预处理：读取图片、解码、尺寸转换、颜色空间转换等。
3. 五类图像检测子模块并行或顺序处理：
   - OCR 文字识别：启动微信 OCR 服务 -> 提交图片识别任务 -> 回调获取识别结果 -> 输出文字内容和位置信息。
   - 二维码检测：读取图片 -> 灰度化、模糊、二值化等预处理 -> OpenCV 二维码检测 -> 输出是否存在二维码、二维码内容和像素位置。
   - 模糊度检测：读取图片 -> 缩放到固定尺寸 -> 灰度化 -> 高斯模糊 -> 拉普拉斯边缘计算 -> 输出边缘强度分数。
   - 色彩丰富程度评估：读取图片 -> 转换 HSV 颜色空间 -> 计算色相熵、饱和度、亮度、对比度 -> 输出综合质量分数和细节指标。
   - 设计感评估：读取图片 -> 图像张量转换与 MobileNet 预处理 -> 加载设计感模型权重 -> 输出平均美学得分和标准差。
4. 图像侧特征汇总：整合 OCR 文本、二维码结果、模糊度分数、色彩分数和设计感分数。
5. 输出到多模态融合分析：与文本风险识别结果共同参与综合风险评分。

建议使用“左侧统一输入，中间五条并列流程，右侧统一汇总输出”的结构：

        ↓

        ↓

        ↓

## 三、五类子模块流程节点说明

| 子模块 | 输入 | 处理流程 | 输出 |
|---|---|---|---|
| OCR 文字识别 | 图片路径 | 自动查找或读取微信 OCR 路径 -> 启动 OCR 服务 -> 提交图片任务 -> 回调保存识别结果 | 图片文字内容、文字位置信息、OCR 结果 JSON |
| 二维码检测 | 图片路径或图片二进制数据 | OpenCV 解码 -> 灰度化 -> 高斯模糊 -> 中值滤波 -> 二值化 -> 二维码检测与解码 | `qr_detected`、`qr_data`、`qr_bbox`、处理过程图 |
| 模糊度检测 | 图片路径 | 图像读取 -> 缩放到 300×300 -> 灰度化 -> 高斯模糊 -> 拉普拉斯边缘计算 | 边缘强度分数，用于反映图片清晰程度 |
| 色彩丰富程度评估 | 图片路径 | 图像读取 -> HSV 转换 -> 色相熵、饱和度、亮度、对比度计算 -> 分数归一化 | 综合质量分数、色彩丰富度、亮度、对比度等细节指标 |
| 设计感评估 | 图片路径 | 图片读取 -> 转换为张量 -> MobileNet 预处理 -> 设计感模型预测 -> 评分统计 | 美学平均分 `mean` 和标准差 `std` |

## 四、OCR 文字识别模块设计细节

对应文件：`picOCR文字识别.py`

### 输入

- 输入数据：本地图片路径
- 依赖环境：微信 OCR 组件
- 配置来源：环境变量 `WECHAT_OCR_EXE`、`WECHAT_DIR`，或自动扫描本机微信 OCR 安装路径

### 处理流程

1. 初始化 OCR 模块。
2. 查找微信 OCR 可执行文件路径和微信安装目录。
3. 创建 OCR 输出目录 `整合/json`。
4. 初始化 `OcrManager`。
5. 设置 OCR 可执行文件路径和用户库目录。
6. 设置 OCR 识别结果回调函数。
7. 启动微信 OCR 服务。
8. 对指定图片执行 OCR 任务。
9. 在回调函数中接收识别结果。
10. 将识别结果保存为 JSON 文件。

### 输出

- 图片中识别到的文字内容
- OCR 识别结果 JSON 文件
- 文字区域位置信息，具体结构由微信 OCR 返回结果决定

### 设计作用

OCR 文字识别用于提取图片中的宣传语、联系方式、金融承诺、收益话术等文本内容。提取后的文字可再次进入文本风险识别流程，从而弥补仅分析帖子正文时可能遗漏图片文字风险的问题。

## 五、二维码检测模块设计细节

对应文件：`pic二维码检测.py`

### 输入

- 输入数据：图片路径 `image_path` 或图片二进制数据 `image_data`
- 读取方式：使用 `np.fromfile` 和 `cv2.imdecode`，兼容中文路径

### 处理流程

1. 读取图片。
2. 获取原始图片宽高。
3. 将图片转换为灰度图。
4. 对灰度图进行高斯模糊。
5. 进行中值滤波。
6. 进行二值化处理。
7. 使用 OpenCV `QRCodeDetector` 检测并解码二维码。
8. 如果检测到二维码，绘制二维码边界框。
9. 返回二维码检测结果和处理过程图。

### 输出

```text
{
  "success": True,
  "original_size": "宽x高",
  "qr_data": 二维码内容或 None,
  "qr_detected": True/False,
  "qr_bbox": 二维码像素位置或 None,
  "processing_steps": {
    "original": 原图,
    "grayscale": 灰度图,
    "gaussian_blur": 高斯模糊图,
    "median_blur": 中值滤波图,
    "binary": 二值化图,
    "detection": 检测框图
  }
}
```

### 设计作用

二维码常用于引流、私域转化、添加客服或跳转外部平台。在网络金融风险舆情中，二维码检测可以辅助识别“诱导扫码”“引流私域”“规避平台监管”等风险特征。

## 六、模糊度检测模块设计细节

对应文件：`pic模糊度检测.py`

### 输入

- 输入数据：图片路径
- 读取方式：`np.fromfile` + `cv2.imdecode`

### 处理流程

1. 读取图片。
2. 将图片缩放为 `300×300`。
3. 转换为灰度图。
4. 使用高斯模糊进行预处理。
5. 使用拉普拉斯算子提取边缘信息。
6. 计算边缘强度均值。

### 输出

- `edge_strength`：图像边缘强度分数

### 设计作用

模糊度检测用于判断图片清晰程度。清晰度过低可能影响 OCR 和二维码识别效果，也可能反映图片经过压缩、遮挡或二次传播。在多模态分析中，该指标可作为图片可信度和可识别性的辅助特征。

## 七、色彩丰富程度评估模块设计细节

对应文件：`pic色彩丰富程度.py`

### 输入

- 输入数据：图片路径
- 颜色空间：HSV

### 处理流程

1. 读取图片。
2. 将 BGR 图片转换为 HSV 颜色空间。
3. 计算色相直方图和色相熵，形成色相得分。
4. 计算饱和度均值和标准差，形成饱和度得分。
5. 计算亮度均值、暗部比例和亮部比例，形成亮度得分。
6. 计算亮度标准差、动态范围和 RMS 对比度，形成对比度得分。
7. 将色彩丰富度、亮度和对比度相加，得到综合质量分数。

### 输出

当 `verbose=True` 时，输出总分和细节指标：

```text
{
  "color_richness_score": 色彩丰富度分数,
  "hue_entropy_score": 色相熵分数,
  "saturation_score": 饱和度分数,
  "brightness_score": 亮度分数,
  "brightness_value": 亮度均值,
  "contrast_score": 对比度分数,
  "contrast_std": 对比度标准差,
  "dynamic_range": 动态范围,
  "dark_ratio": 暗部比例,
  "bright_ratio": 亮部比例,
  "total_score": 综合质量分数
}
```

### 设计作用

色彩丰富程度评估用于衡量图片视觉表现。部分金融营销或诈骗宣传图往往使用高饱和度、强对比、醒目色彩和夸张设计吸引用户注意，因此色彩丰富度、亮度和对比度可作为图像侧风险识别的辅助特征。

## 八、设计感评估模块设计细节

对应文件：`pic设计感.py`

### 输入

- 输入数据：图片路径
- 模型结构：MobileNet 特征提取网络 + Dropout + Dense 评分层
- 权重文件：`整合/weights/mobilenet_weights.h5`

### 处理流程

1. 检查设计感模型权重是否存在。
2. 构建 MobileNet 基础网络。
3. 添加 Dropout 层和 10 类 softmax 评分层。
4. 加载模型权重。
5. 读取图片并转换为数组。
6. 扩展 batch 维度。
7. 执行 MobileNet 预处理。
8. 使用模型预测图片评分分布。
9. 根据评分分布计算平均分和标准差。

### 输出

```text
mean：图片设计感或美学平均分
std：评分标准差
```

### 设计作用

设计感评估用于衡量图片是否具有较强视觉吸引力。网络金融风险内容中，具有明显营销设计感的图片可能更容易吸引用户点击、扫码或参与投资，因此设计感得分可作为宣传包装程度的辅助指标。

## 九、图像侧结果与多模态融合分析的关系

图像内容检测模块并不单独生成最终风险结论，而是为多模态融合分析提供图像侧特征。其输出会与文本风险识别结果共同参与综合风险评估。

在系统中，图像侧特征主要包括：

- OCR 提取文字及位置
- 是否存在二维码
- 二维码内容和像素位置
- 模糊度或边缘强度分数
- 色彩丰富度、亮度、对比度及综合质量分数
- 设计感平均分和标准差

这些指标会进入多模态融合分析模块，与金融属性、风险要素、情感倾向和煽动性得分共同形成最终风险评分和风险解释。
