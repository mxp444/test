## 使用位置

该图用于毕业论文 **3.3.6 后端接口设计** 小节。

图名建议：

**图3-X 后端核心类图**

该类图用于说明系统后端主要类、模块对象和接口处理逻辑之间的静态结构关系。图中不需要展示所有函数，只展示论文中最能体现后端设计结构的核心类和主要方法。

## 图形类型要求

必须生成 UML 类图，不要生成流程图、数据流图或 ER 图。

类图样式参考示例图：

- 每个类使用三层矩形框。
- 第一层：类名。
- 第二层：属性。
- 第三层：方法。
- 属性和方法前可使用 UML 可见性符号：
  - `+` 表示 public
  - `-` 表示 private
  - `#` 表示 protected
- 类与类之间使用连线表示关系。
- 如果表示强包含关系，可使用组合关系。
- 如果表示调用或依赖关系，可使用虚线依赖箭头。
- 使用黑白灰色调。
- 字体使用宋体。
- 字号 14px。
- 画布宽度 590px。
- 图形风格简洁，适合本科毕业论文。
- 不要使用彩色、渐变、阴影和装饰图标。

## 系统背景

系统名称：网络金融舆情风险监测系统。

后端基于 Flask 实现，核心文件位于：

- `整合/backend/app.py`

后端主要职责包括：

- 提供 REST API 接口。
- 管理用户登录和权限。
- 调度多平台爬虫。
- 连接 MongoDB。
- 聚合和序列化舆情数据。
- 调用多模态分析模块。
- 提供后台平台管理、系统设置、数据库管理等接口。

## 类定义说明

### 1. FlaskApp

类名：`FlaskApp`

说明：

抽象表示 Flask 后端应用，负责注册路由、接收前端请求并调用各业务服务。

属性：

- `+ app: Flask`
- `+ mongo_client: MongoClient`
- `+ runtime_config: dict`

方法：

- `+ get_items(): Response`
- `+ get_item_detail(doc_id): Response`
- `+ start_crawl(): Response`
- `+ pause_crawl(): Response`
- `+ analyze_unprocessed(): Response`
- `+ get_admin_database(): Response`

### 2. AuthManager

类名：`AuthManager`

说明：

抽象表示登录、账号和权限管理功能。

属性：

- `- users_file: Path`
- `- current_user: dict`

方法：

- `+ login(username, password): User`
- `+ register(payload): User`
- `+ require_admin(): bool`
- `+ update_profile(payload): bool`
- `+ log_user_activity(action, detail): void`

### 3. MongoRepository

类名：`MongoRepository`

说明：

抽象表示 MongoDB 数据访问层。系统通过它读取四个平台集合、序列化文档和写回分析结果。

属性：

- `- database_name: str`
- `- platform_collections: dict`

方法：

- `+ platform_collection(platform): Collection`
- `+ iter_platform_collections(): Iterator`
- `+ serialize_doc(doc): dict`
- `+ load_analyzed_items(limit): list`
- `+ update_analysis(doc_id, analysis): bool`
- `+ normalize_documents_collections(): void`

### 4. CrawlTask

类名：`CrawlTask`

说明：

表示一次采集任务，负责接收平台列表、启动多平台爬虫、记录日志和统计状态。

属性：

- `- platforms: list`
- `- status: str`
- `- logs: list`
- `- stats: dict`
- `- controller: WebCrawlController`

方法：

- `+ start(platforms): void`
- `+ pause(): void`
- `+ resume(): void`
- `+ append_log(message): void`
- `+ snapshot(): dict`

### 5. WebCrawlController

类名：`WebCrawlController`

说明：

控制爬虫运行过程中的暂停、继续和停止状态。

属性：

- `- paused: bool`
- `- stopped: bool`

方法：

- `+ pause(): void`
- `+ resume(): void`
- `+ poll(): bool`
- `+ wait_if_paused(): void`

### 6. CrawlerRunner

类名：`CrawlerRunner`

说明：

对应 `crawlers/runner.py`，负责根据平台 ID 动态加载对应爬虫入口。

属性：

- `+ platforms: dict`

方法：

- `+ normalize_platforms(platforms): list`
- `+ platform_collection(platform): str`
- `+ run_platform(platform, config): dict`
- `+ crawler_import_context(path): Context`

### 7. PlatformCrawler

类名：`PlatformCrawler`

说明：

抽象表示微博、抖音、百度贴吧、小红书爬虫的共同接口。实际项目中各平台爬虫目录不同，但后端可抽象为统一爬虫接口。

属性：

- `+ platform_id: str`
- `+ keywords: list`
- `+ cookie: str`
- `+ seen_ids: set`

方法：

- `+ run_crawler(config): dict`
- `+ parse_item(raw): dict`
- `+ download_images(item): list`
- `+ save_to_mongo(item): bool`

### 8. AnalysisManager

类名：`AnalysisManager`

说明：

负责加载分析引擎、筛选未分析数据、并行调用分析模块并写回结果。

属性：

- `- analysis_engine_file: str`
- `- max_workers: int`
- `- analyzer: MultimodalRiskFusion`

方法：

- `+ load_analyzer_class(): type`
- `+ analyze_unprocessed(limit): dict`
- `+ analyze_item(item): dict`
- `+ compact_result(result): dict`
- `+ write_analysis_result(item, result): void`

### 9. MultimodalRiskFusion

类名：`MultimodalRiskFusion`

说明：

多模态风险分析统一接口。API 版、本地版、端到端深度网络版都应遵守该接口。

属性：

- `- model_config: dict`

方法：

- `+ analyze(post_text, image_path): dict`
- `+ analyze_batch(items): list`

### 10. AdminDatabaseService

类名：`AdminDatabaseService`

说明：

表示后台数据库管理功能，负责 MongoDB 数据库、集合和文档管理。

属性：

- `- mongo_client: MongoClient`
- `- protected_databases: list`

方法：

- `+ list_databases(): list`
- `+ list_collections(db): list`
- `+ query_documents(db, collection, filter): list`
- `+ insert_document(db, collection, doc): ObjectId`
- `+ update_document(db, collection, doc_id, doc): bool`
- `+ delete_document(db, collection, doc_id): bool`
- `+ bson_to_jsonable(value): object`

### 11. ConfigManager

类名：`ConfigManager`

说明：

负责读取和保存系统运行配置、平台配置和采集源配置。

属性：

- `- runtime_config_file: Path`
- `- platform_settings_file: Path`
- `- collection_sources_file: Path`

方法：

- `+ load_runtime_config(): dict`
- `+ save_runtime_config(data): void`
- `+ load_platform_settings(): dict`
- `+ save_platform_settings(data): void`
- `+ load_collection_sources(): list`

## 类之间的关系

请按以下关系绘制类图。

### 1. FlaskApp 依赖 AuthManager

关系：

```text
FlaskApp ..> AuthManager
```

含义：

Flask 路由需要调用登录、注册和管理员权限校验。

### 2. FlaskApp 依赖 MongoRepository

关系：

```text
FlaskApp ..> MongoRepository
```

含义：

Flask 接口通过数据访问层读取舆情数据和写入分析结果。

### 3. FlaskApp 组合 CrawlTask

关系：

```text
FlaskApp ◆── CrawlTask
```

含义：

后端维护当前采集任务对象。

### 4. CrawlTask 组合 WebCrawlController

关系：

```text
CrawlTask ◆── WebCrawlController
```

含义：

每个采集任务需要控制暂停、继续和状态轮询。

### 5. CrawlTask 依赖 CrawlerRunner

关系：

```text
CrawlTask ..> CrawlerRunner
```

含义：

采集任务通过统一运行器加载具体平台爬虫。

### 6. CrawlerRunner 依赖 PlatformCrawler

关系：

```text
CrawlerRunner ..> PlatformCrawler
```

含义：

运行器根据平台 ID 调用微博、抖音、贴吧、小红书具体爬虫。

### 7. PlatformCrawler 依赖 MongoRepository

关系：

```text
PlatformCrawler ..> MongoRepository
```

含义：

平台爬虫将采集结果写入对应 MongoDB 集合。

### 8. FlaskApp 依赖 AnalysisManager

关系：

```text
FlaskApp ..> AnalysisManager
```

含义：

后端接口调用补分析或自动分析逻辑。

### 9. AnalysisManager 组合 MultimodalRiskFusion

关系：

```text
AnalysisManager ◆── MultimodalRiskFusion
```

含义：

分析管理器持有当前多模态分析引擎实例。

### 10. AnalysisManager 依赖 MongoRepository

关系：

```text
AnalysisManager ..> MongoRepository
```

含义：

分析管理器读取未分析数据，并将结果写回 MongoDB。

### 11. FlaskApp 依赖 AdminDatabaseService

关系：

```text
FlaskApp ..> AdminDatabaseService
```

含义：

后台数据库管理接口由 AdminDatabaseService 处理。

### 12. AdminDatabaseService 依赖 MongoRepository

关系：

```text
AdminDatabaseService ..> MongoRepository
```

含义：

数据库管理服务复用 MongoDB 连接和 BSON 转换逻辑。

### 13. FlaskApp 依赖 ConfigManager

关系：

```text
FlaskApp ..> ConfigManager
```

含义：

系统设置、平台设置和采集源申请都需要读取或保存配置文件。
