建议放置位置：在介绍“系统采用统一调度器管理多平台爬虫，并将不同平台采集结果转换为统一数据结构”之后插入。

## 表格标题

表题建议为：

**表3-X  多平台舆情采集模块设计差异**

其中 X 按论文中第 3 章表格实际顺序填写。

## 格式要求

生成表格时应符合本科毕业论文理工类写作规范，建议采用以下格式：

1. 表题位于表格上方，居中显示。
2. 表序按章节编号，例如“表3-1”“表3-2”。
3. 表格建议采用三线表形式，不使用竖线。
4. 表头文字居中，正文内容可左对齐或居中，长文本尽量压缩为短语。
5. 字体建议使用宋体五号，若表格较宽可适当使用小五号。
6. 表格内容应简洁，不写过长段落，避免影响页面排版。
7. 不展示真实 Cookie、接口密钥、账号密码等敏感信息。
8. 如表格过宽，可适当压缩列名或将“关键配置”和“过滤去重策略”合并表述。

## 各平台真实设计要点

### 微博采集模块

- 平台名称：微博
- 采集入口：微博搜索页 `s.weibo.com`
- 发现方式：根据金融风险关键词构造搜索 URL，结合最近天数或指定时间范围分页抓取
- 采集对象：包含正文、作者、发布时间、图片链接等信息的微博图文内容
- 关键配置：搜索关键词列表、正文筛选关键词、最近采集天数、每个关键词最大页数、是否要求图片、Cookie
- 过滤策略：按关键词命中、发布时间范围、是否包含图片等条件筛选
- 去重策略：使用平台原始微博 ID 生成统一主键，同时维护 `seen_ids.txt`
- 存储集合：`test_weibo`
- 设计特点：以关键词搜索为主，适合捕获公开传播的金融舆情内容；采集结果会被转换为统一字段结构，便于后续多模态分析

### 抖音采集模块

- 平台名称：抖音
- 采集入口：抖音 Web 搜索接口或搜索结果接口
- 发现方式：根据关键词请求搜索结果，优先采集图文类内容
- 采集对象：抖音图文作品，包括标题、正文描述、作者、发布时间、图片等信息
- 关键配置：搜索关键词列表、每个关键词最大页数、内容类型参数、Cookie、图片下载目录
- 过滤策略：优先保留包含图片和文本描述的作品，按关键词进行内容筛选
- 去重策略：使用抖音作品 ID 作为主键，并维护 `seen_ids.txt`
- 存储集合：`test_douyin`
- 设计特点：依赖平台搜索接口与登录态配置，适合补充短视频/图文平台中的金融推广类舆情数据

### 百度贴吧采集模块

- 平台名称：百度贴吧
- 采集入口：指定贴吧主题帖列表页
- 发现方式：先配置多个相关贴吧名称，再在不同贴吧之间随机轮换抓取主题帖
- 采集对象：贴吧主题帖及其正文、作者、发布时间、图片等信息
- 关键配置：贴吧名称列表、正文筛选关键词、每个贴吧最大页数、是否随机切换贴吧、是否要求图片
- 过滤策略：遇到受限或不可访问贴吧时跳过；按关键词、图片和内容完整性筛选
- 去重策略：使用帖子 ID 生成统一主键，并维护 `seen_ids.txt`
- 存储集合：`test_baidu`
- 设计特点：采用“贴吧轮换”策略，避免长时间集中访问单一贴吧；适合采集社区讨论型金融风险舆情

### 小红书采集模块

- 平台名称：小红书
- 采集入口：小红书关键词搜索与笔记详情接口
- 发现方式：根据关键词检索笔记，再获取笔记详情
- 采集对象：小红书图文笔记，包括标题、正文、作者、标签、图片、发布时间等信息
- 关键配置：关键词列表、每个关键词最大页数、笔记类型、Cookie 或 `.env` 登录配置
- 过滤策略：优先保留图文笔记，按关键词、图片、正文完整性进行筛选
- 去重策略：使用 `note_id` 生成统一主键，并维护 `seen_ids.txt`
- 存储集合：`test_xhs`
- 设计特点：对笔记内容进行结构化解析后转换为系统统一字段，便于与微博、抖音、贴吧数据共同展示与分析

## 统一数据结构说明

虽然四个平台的网页结构、接口形式和内容组织方式不同，但系统在入库前会将采集结果转换为相近的数据字段，主要包括：

- `id`：系统统一主键
- `platform`：平台标识
- `platform_name`：平台中文名称
- `screen_name`：作者或账号名称
- `text`：正文内容
- `created_at`：发布时间
- `keyword`：采集关键词
- `matched_keywords`：命中的风险关键词
- `pics`：图片路径或图片链接
- `crawl_time`：采集时间
- `analysis_status`：分析状态

该统一结构保证了后续 MongoDB 存储、前端展示、多模态分析和风险筛选逻辑能够复用同一套处理流程。
