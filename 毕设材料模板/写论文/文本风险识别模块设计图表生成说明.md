## 一、文本风险识别子模块工作流程图设计

### 流程图核心逻辑

文本风险识别模块以舆情正文 `text` 作为统一输入。为了体现模块设计细节，流程图应重点展示四类文本分析子模块各自的工作路径：

1. 统一输入：舆情正文文本 `text`。
2. 文本预处理：空值判断、长度截断、模型编码或分词准备。
3. 四类文本分析子模块并行处理：
   - 金融属性判断：BERT 编码 -> 金融分类模型预测 -> 输出前 4 个金融类别及置信度。
   - 风险要素识别：`jieba` 分词 -> 词向量相似度计算 -> 风险类别归属 -> 输出风险词、类别统计和总风险词数。
   - 情感分析：BERT 编码 -> FinBERT 五分类预测 -> softmax 置信度计算 -> 输出情感标签、风险等级和置信度。
   - 煽动性评估：规则词库与正则匹配 -> 语气、稀缺性、从众压力三维度评分 -> 输出总分、风险等级和命中细节。
4. 文本风险特征汇总：整合金融属性、风险词统计、情感倾向和煽动性结果。
5. 输出到多模态融合分析：作为文本侧风险特征参与综合评分。

### 流程图建议结构

建议使用“左侧统一输入，中间四条并列流程，右侧统一汇总输出”的结构：

```text
舆情正文 text
     ↓
文本预处理
     ↓
┌──────────────────────────────────────────────┐
│  金融属性判断：BERT编码 -> 分类预测 -> 前4类及置信度  │
│  风险要素识别：分词 -> 词向量相似度 -> 风险词与类别统计 │
│  情感分析：BERT编码 -> 五分类预测 -> 情感标签与置信度   │
│  煽动性评估：规则匹配 -> 三维度评分 -> 总分与风险等级   │
└──────────────────────────────────────────────┘
     ↓
文本风险特征汇总
     ↓
输出至多模态融合分析
```

### 四类子模块流程节点说明

| 子模块 | 输入 | 处理流程 | 输出 |
|---|---|---|---|
| 金融属性判断 | 舆情正文 `text` | 文本编码 -> 本地金融 BERT 分类模型预测 -> 概率排序 | 前 4 个金融类别及置信度 |
| 风险要素识别 | 舆情正文 `text` | `jieba` 分词 -> 金融词向量加载 -> 计算词语与风险类别向量相似度 -> 按阈值归类 | 风险类别、命中词、相似度、各类别词数、总风险词数 |
| 情感分析 | 舆情正文 `text` | BERT 编码 -> FinBERT 五分类模型预测 -> softmax 计算置信度 | 情感标签、标签编号、风险等级、置信度 |
| 煽动性评估 | 舆情正文 `text` | 规则词库和正则匹配 -> 语气强烈程度评分 -> 稀缺性暗示评分 -> 从众压力评分 | 三个维度分数、总分、风险等级、风险描述、处理建议 |

## 三、金融属性判断模块设计细节

对应文件：`nlp金融属性判断.py`

### 输入

- 输入字段：舆情正文 `text`
- 文本最大长度：模型编码时使用 `max_length=512`
- 模型目录：默认使用本地 `my_finance_bert_wwm`
- 加载方式：本地加载 `BertTokenizer`、`BertForSequenceClassification` 和 `label_encoder.pkl`

### 方法

该模块基于训练好的金融文本分类模型，对输入文本进行多类别分类预测。模型输出每个标签的概率，系统取概率最高的前 4 个标签作为结果。

### 输出

输出格式为金融类别及其置信度列表：

```text
[
  (类别1, 置信度1),
  (类别2, 置信度2),
  (类别3, 置信度3),
  (类别4, 置信度4)
]
```

### 金融类别

模型涉及 12 类金融舆情标签：

| 编号 | 类别含义 |
|---|---|
| 0 | 非金融舆情类但胜似金融舆情，如招商、合伙人、返利、层级分红 |
| 1 | 完全非金融舆情类，如生活、日常、兴趣 |
| 2 | 金融诈骗话术，如高收益、保本兜底、拉下线、庞氏 |
| 3 | 理财经理产品推销，如正规理财、基金、存款、信托 |
| 4 | 客户视角理财分享，如个人理财经历、收益、产品 |
| 5 | 金融避雷踩坑吐槽，如被骗、踩坑、提醒 |
| 6 | A 股股价走势分析，如大盘、板块、个股、行情 |
| 7 | 金融政策监管动态解读，如央行、监管局、政策 |
| 8 | 基金定投相关讨论，如定投技巧、误区、方法 |
| 9 | 保险产品测评讨论，如保险选购、险种、建议 |
| 10 | 期货贵金属投资讨论，如黄金、原油、期货、风险 |
| 11 | 数字货币相关舆情，如比特币、监管、虚拟币 |

## 四、风险要素识别模块设计细节

对应文件：`nlp风险要素识别.py`

### 输入

- 输入字段：舆情正文 `text`
- 分词工具：`jieba`
- 词向量模型：`sgns.financial.word`

### 方法

模块预先设置 16 类金融风险种子词，并基于金融词向量模型构建各风险类别向量。文本输入后先进行中文分词，再逐词计算与各风险类别向量的余弦相似度。当最高相似度大于阈值 `0.40` 时，将该词归入对应风险类别。

### 主要风险类别

| 序号 | 风险类别 |
|---|---|
| 1 | 保本无风险虚假承诺类 |
| 2 | 超高收益诱惑类 |
| 3 | 传销拉人头层级返利类 |
| 4 | 内幕消息违规荐股类 |
| 5 | 紧迫饥饿营销逼单类 |
| 6 | 虚假资质权威背书类 |
| 7 | 虚拟货币数字资产诈骗类 |
| 8 | 征信修复贷款诈骗类 |
| 9 | 私域引流联系方式类 |
| 10 | 虚假造势从众诱导类 |
| 11 | 非法外盘跨境金融交易类 |
| 12 | 养老金融诈骗类 |
| 13 | 资金盘庞氏骗局特征类 |
| 14 | 代客理财违规操作类 |
| 15 | 数字藏品元宇宙金融骗局类 |
| 16 | 股票期货配资违规交易类 |

### 输出

输出包括两个部分：

1. `result`：各风险类别下命中的词语及相似度得分。
2. `summary`：各风险类别命中数量和总风险词数。

示例结构：

```text
{
  "保本无风险虚假承诺类": [{"word": "保本", "score": 0.8321}],
  "超高收益诱惑类": [{"word": "高收益", "score": 0.7952}],
  ...
}

{
  "总风险词数": 25,
  "传销拉人头层级返利类": 6,
  "超高收益诱惑类": 4,
  ...
}
```

## 五、情感分析模块设计细节

对应文件：`nlp情感分析.py`

### 输入

- 输入字段：舆情正文 `text`
- 模型输入最大长度：`max_length=128`
- 模型结构：基于 BERT 的五分类模型 `FinBERT5Class`
- 权重文件：`finbert_5class_best.pth`

### 方法

模块使用金融语境下的情感分类模型，对文本进行五分类预测。模型通过 softmax 输出各情感类别概率，并选择概率最高的类别作为最终情感标签。

### 情感标签

| 标签 ID | 情感标签 | 对应风险等级 |
|---|---|---|
| 0 | 中性 | 无风险 |
| 1 | 负面-恐慌/恶意唱空（高风险） | 高风险 |
| 2 | 负面-理性担忧（低风险） | 低风险 |
| 3 | 正面-理性看好（低风险） | 低风险 |
| 4 | 正面-过度狂热（高风险） | 高风险 |

### 输出

```text
{
  "text": 原始文本,
  "sentiment": 情感标签,
  "label_id": 标签编号,
  "risk_level": 风险等级,
  "confidence": 置信度
}
```

## 六、煽动性评估模块设计细节

对应文件：`nlp煽动性评估.py`

### 输入

- 输入字段：舆情正文 `text`
- 方法类型：规则词库 + 正则模式 + 分维度评分

### 评估维度

| 维度 | 满分 | 检测内容 | 设计含义 |
|---|---:|---|---|
| 语气强烈程度 | 40 | 感叹号、连续标点、全大写比例、极端词汇 | 识别夸张、激烈、情绪化的表达 |
| 稀缺性暗示 | 30 | 限时、限量、最后机会、名额紧张、倒计时等 | 识别逼单式、制造紧迫感的话术 |
| 从众压力 | 30 | 万人参与、大家都在买、成功案例、好评反馈等 | 识别利用群体效应诱导决策的话术 |

### 风险等级

| 总分范围 | 风险等级 | 说明 |
|---|---|---|
| 80 分及以上 | 极高风险 | 文本具有极强煽动性，可能涉及诈骗或恶意营销 |
| 60-79 分 | 高风险 | 文本煽动性明显，需要警惕误导 |
| 40-59 分 | 中等风险 | 文本具有一定煽动性，建议谨慎对待 |
| 20-39 分 | 低风险 | 文本煽动性较弱，基本属于正常表达 |
| 0-19 分 | 无风险 | 文本客观理性，无明显煽动性 |

### 输出

输出包括总分、风险等级、风险描述、各维度得分、百分比、命中细节和处理建议：

```text
{
  "total_score": 总分,
  "risk_level": 风险等级,
  "risk_description": 风险描述,
  "dimensions": {
    "tone": {"score": 分数, "max_score": 40, "percentage": 百分比, "details": 命中细节},
    "scarcity": {"score": 分数, "max_score": 30, "percentage": 百分比, "details": 命中细节},
    "herd": {"score": 分数, "max_score": 30, "percentage": 百分比, "details": 命中细节}
  },
  "recommendation": 处理建议
}
```

## 七、与多模态融合分析的关系

文本风险识别模块并不是独立生成最终风险结论，而是为多模态融合分析提供文本侧特征。其输出会与图片内容检测、OCR 文字识别、二维码检测、模糊度分析、色彩丰富度分析和设计感分析等图像侧结果共同参与综合风险评估。

在系统实现中，最终多模态分析结果会写入 MongoDB 文档的 `analysis` 字段，并通过前端详情报告展示。文本风险识别结果主要用于解释“为什么该帖子存在风险”，提高系统风险预警的可解释性。

## 十、建议在论文中的写法重点

撰写 3.3.3 节时可以围绕以下几点展开：

1. 文本风险识别模块位于采集模块之后、多模态融合模块之前。
2. 模块输入为统一舆情文档中的正文文本，也可结合 OCR 识别文字补充分析。
3. 金融属性判断用于识别文本所属金融语义类别。
4. 风险要素识别用于提取具体风险词和风险类别，提高解释性。
5. 情感分析用于判断金融语境下的恐慌、理性担忧、理性看好或过度狂热。
6. 煽动性评估用于发现限时、保本、高收益、从众诱导等营销话术。
7. 四类文本结果共同构成文本侧风险特征，为最终多模态风险评分提供依据。
